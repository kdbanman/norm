ADR-001 — Encrypted index: AES-256-GCM over serialized SQLite (not SQLCipher)

Affects: REQ-INIT-001 REQ-SEC-001 REQ-SEC-007 · concept §6, §7

Context. The concept/requirements named SQLCipher for the index. SQLCipher has no installable build for this target: no pip wheel for arm64/CPython 3.12 (only manylinux_x86_64), and it is not present via Homebrew. The realistic alternative was brew install sqlcipher + a source build of the sqlcipher3 binding on every dev/CI machine.

Decided The index is an ordinary SQLite database held in memory only while a command runs; at rest it is Connection.serialize() → AES-256-GCM → data_dir/index.db, and opened via the reverse (deserialize()). Blob and key-wrapping crypto are unchanged (AES-256-GCM blobs; Argon2id-wrapped data key).

Pure Python (cryptography + stdlib sqlite3), zero system dependencies. The security contract is satisfied unchanged: the on-disk file carries no SQLite format 3\0 header and is unreadable without the data key (REQ-SEC-001). Trade-off vs. SQLCipher's page-level encryption: the whole index is decrypted into process memory while unlocked — acceptable for norm's short-lived commands and its threat model, which already admits in-memory plaintext during operation, and the index holds metadata only (blobs stay out of it). Approved by the maintainer.
