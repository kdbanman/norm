19 — Verifying the Keychain data key and its ACL scoping

Verification approach Resolved
Affects: REQ-INIT-001 (key exists), REQ-SEC-002 (distinct from login password); security model §7

The gap

"A norm data key exists in the Keychain with ACL scoped to the norm binary" is checked via the Keychain, not via norm's stdout. Existence is easy to assert; ACL scoping is genuinely hard to verify from outside.

Options — three distinct checks

1. Existence (easy, automatable)

Automatable security find-generic-password -s "norm:datakey" returns success after init and absence before. Assert the item is present, is a generic-password item, and was created by norm.

2. Randomness / distinctness from login password

Argued by construction Assert key material is 256-bit and is not equal to or derived from the login password (supports REQ-SEC-002). Because the raw bytes aren't meant to be extractable, this is mostly verified by code review of the key-generation path rather than a runtime assertion.

3. ACL scoping (hard)

Fully asserting "only the norm binary may read it" likely needs a Keychain-API harness or manual inspection.

Practical proxy Delete or withhold the Keychain item and assert norm then reports the locked state (exit 3), closing the loop with REQ-SEC-004. This proves the key is the gate, even if it doesn't prove the ACL boundary.

Recommendation: automate existence + the locked-without-key proxy; flag the ACL boundary as a manual / code-review acceptance item (see [20]).

Resolution

Decided Obsolete — norm uses no macOS Keychain; the access gate is the app password (key model in [25], security model §7). Replace the three Keychain checks with: (1) Existence — after init, assert data_dir/key.json (the Argon2id-wrapped data key) exists and that no norm Keychain item exists (security find-generic-password -s "norm:datakey" finds nothing). (2) Distinctness — the wrapped key is password-derived and never equal to/derived from the login password (code review of the key-gen path, REQ-SEC-002). (3) "Is it the gate" proxy — withhold the password and assert norm reports locked (exit 3, STORE_LOCKED), closing the loop with REQ-SEC-004. ACL-binding-to-a-signed-binary is no longer relied upon, so the hard-to-verify ACL item is dropped entirely.
