14 — Complete CLI surface + flag ↔ config-key mapping

Spec clarification Resolved
Affects: REQ-GLOBAL-005 (precedence), every command; conventions config.keys, global_flags

What's unclear

The examples reference many flags, but there is no complete per-command flag list, and the mapping between CLI flags and config keys is partial and named inconsistently (--interval ↔ interval_minutes, --window ↔ window_k, --stride ↔ stride_j, --idle-threshold ↔ idle_threshold_seconds). Unknowns: do --model, --data-dir, --max-tokens, --phash-threshold exist as flags? Several config keys have no flag and no requirement: max_tokens, phash_threshold, data_dir, prompt_interval. Are they CLI-overridable? Tested?

Why it matters

Goal #1 (comprehensive CLI assertions) needs the full flag inventory and the precedence rule (CLI flag > ~/.norm > default) testable for every overridable key, not just interval_minutes (the only one in REQ-GLOBAL-005).

Suspected mapping (to confirm / complete)

Config key CLI flag Commands Has requirement?

idle_threshold_seconds --idle-threshold record RECORD-002

Decision: confirm which keys are flag-overridable, name the flags canonically, and add precedence requirements — or explicitly mark them "config-only."

Resolution

DecidedEvery config key is flag-overridable, with uniform precedence CLI flag > ~/.norm > built-in default. Canonical flags:

  • interval_minutes → --interval; idle_threshold_seconds → --idle-threshold; data_dir → --data-dir; phash_threshold → --phash-threshold; model → --model; window_k → --window; stride_j → --stride; max_tokens → --max-tokens; prompt_preprocess → --prompt-preprocess; prompt_interval → --prompt-interval.
  • Non-config command flags: --once, --install/--status/--stop (record); --force (preprocess); --auto-preprocess/--strict (interval); --from/--to/--last/--before; --output; --include; --export; --dry-run; --passphrase; globals --config/--json/--verbose/--quiet/--help/--version.
  • Precedence is testable for every overridable key, generalizing REQ-GLOBAL-005.

Rationale: the user wants a flag for every setting; uniform precedence is the most flexible and the most testable.
