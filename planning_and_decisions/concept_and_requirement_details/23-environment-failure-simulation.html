23 — Simulating environment failures (permissions, missing deps, offline)

Verification approach Resolved
Affects: REQ-RECORD-007 (no permission → 6), REQ-ENV-001 (macapptree missing → 6), REQ-PREPROCESS-005 (model unavailable offline → 4)

The gap

Three exit codes depend on environmental fault states that are awkward to produce on a healthy dev machine: revoked TCC permissions, an absent capture dependency, and offline-with-uncached-weights. Without a way to induce them on demand, these codes can't be tested reliably.

Per-fault approach

Missing macOS permissions → exit 6 (RECORD-007)

tccutil reset ScreenCapture / Accessibility works but does real OS-state surgery.

Suspected, more portable Fault-injection seam NORM_FORCE_NO_PERMISSION=screen|ax — forces the permission probe to report "denied," exercising the error path and message without touching TCC. Keep one real-environment smoke test per permission type.

Missing macapptree → exit 6 (ENV-001)

Suspected Force its import to fail via NORM_FORCE_NO_MACAPPTREE=1, or shadow it on PYTHONPATH with a stub module that raises ImportError. Assert exit 6 and that the message names the dependency.

Model unavailable offline → exit 4 (PREPROCESS-005)

Suspected Two sub-cases to cover separately:
  • Weights unavailable: redirect HF_HOME to an empty directory and block network (or set HF offline mode).
  • Invalid model_ref: pass a nonsense string as --model. Assert exit 4 and that the message distinguishes the two causes where possible.

Principle

Prefer fault-injection env seams over real OS-state mutation so negative tests are hermetic and CI-safe. Keep one real-environment smoke test per fault code to confirm the seam faithfully represents the real path.

Resolution

Decided Prefer hermetic fault-injection env seams over real OS-state mutation, keeping one real smoke test per fault. NORM_FORCE_NO_PERMISSION=screen|ax forces the permission probe to report denied → exit 6 with a remediation message (REQ-RECORD-007); keep one tccutil-based smoke test. NORM_FORCE_NO_MACAPPTREE=1 (or a PYTHONPATH stub raising ImportError) → exit 6 naming the dependency (REQ-ENV-001). Model-unavailable splits cleanly now that download is init-only: "weights not provisioned" = skip init / --skip-model (or empty HF_HOME) → exit 4, MODEL_UNAVAILABLE, message "run norm init"; "invalid model_ref" = nonsense --model → exit 4, INVALID_MODEL_REF (REQ-PREPROCESS-005). Every message names the problem and the fix.
