21 — Verifying "no elevated privileges / no privileged helper"

Verification approach Unresolved
Affects: REQ-SEC-003, REQ-RECORD-009 (--install requires no sudo)

The gap

"Every command works as a normal user; no setuid helper or privileged daemon installed" is a system-state property verified by auditing process and filesystem state, not by norm's output.

Options (executable, non-CLI)

Suspected core Run-as-user, no-sudo harness — execute the full command suite as an unprivileged user with no sudo available on PATH. Assert each command either succeeds or fails for its documented reason — never a privilege error.

Post-run filesystem audits

  • find the norm install and any files it created; assert none have setuid/setgid bits set.
  • Assert record --install creates a LaunchAgent (~/Library/LaunchAgents/…, user domain) and not a LaunchDaemon (/Library/LaunchDaemons, root). Verify with launchctl print gui/<uid>/<label> vs. the system domain.
  • Assert no root-owned files appear under data_dir or the install location.

--install without sudo

The launchd registration should succeed in the user GUI domain without elevated privileges. Confirm via --status showing running=true with a pid owned by the current user.
