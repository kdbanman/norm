22 — Making capture gating (idle + dedupe) deterministically testable

Verification approach Resolved
Affects: REQ-RECORD-001, REQ-RECORD-002 (idle), REQ-RECORD-003/004 (dedupe); depends on the dedupe hash-input definition in norm-concept §8

The gap

The idle gate reads real HIDIdleTime, and dedupe depends on real on-screen pixels and the live AX tree. To assert "stored when present and changed," "skipped while idle," "deduped when unchanged," and "stored when only one of phash/AX changes," a test must control both the idle value and the captured frame/AX content — neither of which is a CLI input today.

Options

Suspected best (a) Injection seam — env/test hooks that feed a scripted idle value (e.g. NORM_FAKE_IDLE=180) and a scripted frame + AX source (NORM_FAKE_CAPTURE=<dir> of prepared image/AX pairs). Makes record --once fully deterministic and lets us construct the exact phash/AX combinations REQ-RECORD-003/004 require. Pairs with the hash-input definition in norm-concept §8.
  • (b) Real-environment drive — leave the machine idle for N seconds for the idle test; change/leave the screen for dedupe. Slow, flaky, and hard in CI; useful as an occasional end-to-end smoke test only.
  • (c) Below-the-CLI unit tests — test the gate/dedupe functions directly. Strong and fast, but not a black-box CLI assertion. Acceptable under goal #2; complements (a).

Recommendation: (a) for CLI-level acceptance tests + (c) for the hash/threshold math.

Resolution

Decided Adopt (a) + (c). Hidden injection seams make record --once deterministic: NORM_FAKE_IDLE=<seconds> feeds a scripted HIDIdleTime, and NORM_FAKE_CAPTURE=<dir> supplies prepared image + AX pairs in place of macapptree, letting tests construct the exact phash/AX combinations for REQ-RECORD-001/002/003/004 (stored-when-changed, skipped-while-idle, deduped-when-unchanged, stored-when-only-one-of-phash/AX-changes). The hash/threshold math is also unit-tested directly against the AX normalization in [08]. These remain test-only seams, not product features.
